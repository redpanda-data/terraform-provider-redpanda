# CLAUDE.md

This file guides Claude Code in the terraform-provider-redpanda repository.

## Essential Commands

Task runner: [Task](https://taskfile.dev). Use `task` directly (no wrapper).

### Pre-commit / quality
- `task ready` ‚Äî docs + lint + `go mod tidy` (run before every commit)
- `task lint` / `task lint:fix`
- `task build:tidy` ‚Äî `go mod tidy` only

### Tests
- `task test:unit` ‚Äî race-detector unit tests, no cloud creds
- `task test:cluster:aws` / `:gcp` / `:azure` ‚Äî live cluster acc tests
- `task test:byoc:aws` / `:gcp` / `:azure` ‚Äî BYOC acc tests
- `task test:byovpc:aws` ‚Äî provisions infra, runs test, tears down
- `task test:serverless:aws:public` / `:aws:private` / `:aws:both` / `:gcp` / `:regions` / `:privatelink`
- `task test:datasource:cluster` ‚Äî datasource acc test (creates cluster + reads)
- `task test:network` ‚Äî network resource acc test

Live acc tests require `REDPANDA_CLIENT_ID` + `REDPANDA_CLIENT_SECRET` and cloud-provider creds.

### Cleanup (for stuck/leaked resources)
- `task cleanup:aws:ci` ‚Äî nuke BYOVPC AWS resources (auto-approve, CI-safe)
- `task cleanup:aws` ‚Äî same but interactive
- `task cleanup:aws:nuke` ‚Äî nuke all non-default VPCs in test regions (manual, last resort)
- `task cleanup:redpanda` ‚Äî delete stale `tfrp-*` clusters from Redpanda Cloud
- `task cleanup:redpanda:dry` ‚Äî preview only

### Codegen / docs
- `task generate:models` ‚Äî regenerate `redpanda/models/*/*_gen.go` from schema AST
- `task generate:clean` ‚Äî delete generated model files
- `task docs` ‚Äî regenerate `docs/` via tfplugindocs
- `task mock` ‚Äî regenerate `redpanda/mocks/*`

### Build / install / release
- `task build` ‚Äî build `terraform-provider-redpanda` binary
- `task build:install` ‚Äî install to local Terraform plugin cache
- `task local:cluster:aws:apply` / `:destroy` ‚Äî apply an example stack locally

## Repository Architecture

- `redpanda/redpanda.go` ‚Äî provider entry: `New`
- `redpanda/cloud/` ‚Äî gRPC client, `connpool`, `ratelimiter`, `controlplane.go`
- `redpanda/models/` ‚Äî Terraform state ‚áî proto conversion. `*_gen.go` files are produced by codegen; do not hand-edit
- `redpanda/resources/<resource>/` ‚Äî resource + datasource implementations (cluster, network, topic, acl, user, role, role-assignment, pipeline, schema, schemaregistryacl, serverlesscluster, serverlessprivatelink, serverlessregions, region, regions, resourcegroup, throughputtiers)
- `redpanda/tests/` ‚Äî live acc tests; `tests/infra/` holds Terraform stacks used by BYOVPC tests
- `redpanda/mocks/` ‚Äî gomock-style client mocks (unit tests)
- `redpanda/utils/` ‚Äî shared helpers (retry, backoff, etc.)
- `pkg/generate/modelgen/` ‚Äî AST-based model code generator
- `scripts/cleanup-redpanda-byovpc/` ‚Äî BYOVPC sweeper binary (used by `task cleanup:aws:*`)
- `docs/` ‚Äî generated by tfplugindocs; regenerate with `task docs`
- `examples/` ‚Äî example `.tf` configs referenced by docs
- `templates/` ‚Äî tfplugindocs templates

## Testing

Two tiers. Pick the narrowest that exercises the behavior.

| Tier | Where | When to use | Creds |
|------|-------|-------------|-------|
| Unit | `*_test.go` next to the code | Pure logic, mapping, validation; uses `redpanda/mocks/` gomock clients | None |
| Live acc | `redpanda/tests/*_test.go` via `task test:<scope>` | Real cluster lifecycle, BYOC, BYOVPC, end-to-end flows | `REDPANDA_CLIENT_ID/SECRET` + cloud-provider creds |

Prefer unit tests with gomock clients for anything that can be exercised in-process. Reach for live acc only when the behavior genuinely requires a real cluster.

## Code Generation

Models: run `task generate:models` after editing a schema in `redpanda/resources/<resource>/schema_*.go`. Review the `*_gen.go` diff before committing.

Golden files (`*.golden`) and `.description` files are **sacred** ‚Äî never modify without explicit user approval. They pin contracts that drift silently without these guardrails.

Never swallow warnings or errors from codegen. Surface them.

## Documentation

`task docs` regenerates `docs/` from schema descriptions + `examples/` via tfplugindocs. Never hand-edit `docs/*.md` ‚Äî they will be overwritten.

If you change a schema `Description` field, run `task docs` and commit the diff in the same commit.

## Git & PR Workflow

Never use `git checkout`, `git restore`, or `git reset --hard` to discard files without explicit user instruction. The user may have intentional in-progress work.

Before every commit: `task ready` (or at minimum `task lint`).

### GitHub PR comments
- All comments on PR: `gh api repos/redpanda-data/terraform-provider-redpanda/pulls/<N>/comments --paginate`
- Single comment by ID: `gh api repos/redpanda-data/terraform-provider-redpanda/pulls/comments/<ID>`
- Comment ID comes from the URL fragment: `#discussion_r2382392371` ‚Üí ID `2382392371`

## Conventions

- Comments: short, functional, essential. No background prose, no narration, no multi-paragraph explanations.
- Never add `//nolint`, `#nosec`, or any linter suppression without explicit user approval.
- `allow_deletion=false` in cluster tests is intentional and acts as a canary for testing failures. Fix the upstream failure, don't flip the flag.
- When reporting/fixing bugs: write a failing test first, show the red, *then* discuss fix scope separately.
- Clarifying questions from the user ("wait, is that correct?") are not pushback. Verify and answer ‚Äî don't withdraw a correct finding.
- Never delete files or large blocks of code without summarizing and asking first.

## Troubleshooting

- **Acc test hangs or leaves dangling cloud resources**: kill the test process, then `task cleanup:aws:ci` (plus `task cleanup:redpanda` for stale clusters). Don't wait for the test framework to destroy ‚Äî it often can't.
- **Lint fails after codegen**: `task lint:fix`, then review the diff.
- **Docs CI fails**: you changed a schema description; run `task docs` and commit the regenerated files.
- **Local provider testing**: `task build:install` copies the binary into `.terraform.d/plugins/‚Ä¶`; point your `.terraformrc` at that dev override to consume the local build.
