# User and ACLs

Provisions a SASL/SCRAM user and a typical consumer-style ACL bundle
against an **existing** Redpanda Cloud cluster (BYOC or Dedicated).

The cluster, resource group, and network are assumed to already exist —
this example only manages the user and its permissions. If you also need
to provision the cluster from scratch, see
[`examples/redpanda-role`](../redpanda-role) for a full bootstrap.

## What gets created

- 1× `redpanda_user` — SASL/SCRAM-SHA-256 with a random 24-character
  password generated by `random_password` and stored as a write-only
  attribute (`password_wo`), so it never lands in Terraform state.
- 2× `redpanda_acl` on the `CLUSTER` resource — `READ`, `DESCRIBE`.
- 3× `redpanda_acl` on `TOPIC *` — `DESCRIBE`, `WRITE`, `READ`.
- 4× `redpanda_acl` on `GROUP *` — `READ`, `CREATE`, `WRITE`,
  `DESCRIBE`.

The ACL set uses `for_each` over operation lists declared in `locals`,
which is the pattern most production stacks settle on once they outgrow
hand-rolled ACL blocks. Add or remove operations by editing the lists in
`main.tf` — no other changes required.

## Prerequisites

- Terraform `>= 1.11` (required for the write-only `password_wo`
  attribute).
- An existing Redpanda Cloud cluster (BYOC or Dedicated). You need its
  dataplane API URL, which has the form
  `https://api-<id>.<rg>.byoc.prd.cloud.redpanda.com`. It is shown on
  the cluster's details page in the Redpanda Cloud UI, and is also
  returned as `dataplane_api.url` by
  [`GET /v1/clusters/{id}`](https://docs.redpanda.com/api/doc/cloud-controlplane/operation/operation-clusterservice_getcluster)
  on the Control Plane API.

- A Redpanda Cloud **service account with the `Admin` role**, used by
  the Terraform provider to authenticate. Create one via the
  [Control Plane RBAC API](https://docs.redpanda.com/redpanda-cloud/manage/api/cloud-byoc-controlplane-api/#manage-rbac)
  or **Organization IAM → Service accounts** in the Redpanda Cloud UI,
  then assign the `Admin` role to it.

  > For details on predefined roles (`Reader`, `Writer`, `Admin`) and
  > how to scope them, see [RBAC in Redpanda Cloud](https://docs.redpanda.com/redpanda-cloud/security/authorization/rbac/rbac/#predefined-roles).
  > The `Admin` role grants the data plane permissions required to
  > create users and ACLs on the cluster.

  When you create the service account you'll receive a `client_id` and
  `client_secret`. The Terraform provider reads them from the
  `REDPANDA_CLIENT_ID` and `REDPANDA_CLIENT_SECRET` environment
  variables.

## Run

```sh
export REDPANDA_CLIENT_ID=<service-account-client-id>
export REDPANDA_CLIENT_SECRET=<service-account-client-secret>

terraform init
terraform apply \
  -var 'cluster_api_url=https://api-<id>.<rg>.byoc.prd.cloud.redpanda.com'
```

Override the username with `-var 'user_name=my-user'` if you want
something other than the default `example-user`.

## Outputs

| Output | Description |
| --- | --- |
| `user_name` | The name of the created user (echoes `var.user_name`). |

The user's password is intentionally not exported — it's stored as a
write-only attribute and never written to state. If you need the
password for downstream tooling, sink it into your secrets backend at
apply time using a separate resource (e.g. `aws_secretsmanager_secret`,
`vault_kv_secret_v2`) and reference `random_password.user.result`.

## Cleanup

```sh
terraform destroy \
  -var 'cluster_api_url=https://api-<id>.<rg>.byoc.prd.cloud.redpanda.com'
```

`allow_deletion = true` is set on every resource so `destroy` will
remove them cleanly.

## Customizing the ACL set

The operation lists in `locals` are the entire surface area for
adjustment:

```hcl
locals {
  cluster_ops = ["READ", "DESCRIBE"]
  topic_ops   = ["DESCRIBE", "WRITE", "READ"]
  group_ops   = ["READ", "CREATE", "WRITE", "DESCRIBE"]
}
```

To narrow the user to a specific topic prefix rather than `*`, change
`resource_name` on `redpanda_acl.topic` and set `resource_pattern_type`
to `"PREFIXED"`. See the
[`redpanda_acl` resource docs](https://registry.terraform.io/providers/redpanda-data/redpanda/latest/docs/resources/acl)
for the full schema.
