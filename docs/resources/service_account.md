# redpanda_service_account (Resource)

A Redpanda Cloud service account (machine identity)

Creates a Redpanda Cloud service account — a machine identity used by automated workloads (CI pipelines, operators) to authenticate against the Redpanda Cloud Control Plane API.

## Client Secret Handling

The `auth0_client_credentials.client_secret` is generated by the server and returned only in the Create response. Subsequent reads cannot refresh it, so the provider captures the value in Terraform state on creation and treats the field as `Sensitive`.

- Imported service accounts cannot recover their original `client_secret` — the field will be empty in state after `terraform import`.
- Treat your Terraform state as confidential when service accounts are managed by this resource (use a remote backend with encryption at rest).

## Example Usage

```terraform
provider "redpanda" {}

resource "redpanda_service_account" "example" {
  name        = "ci-bot"
  description = "Service account used by CI workloads."
}

output "client_id" {
  value = redpanda_service_account.example.auth0_client_credentials.client_id
}

output "client_secret" {
  value     = redpanda_service_account.example.auth0_client_credentials.client_secret
  sensitive = true
}
```

## Import

```shell
terraform import redpanda_service_account.example serviceAccountId
```

Where `serviceAccountId` is the 20-character identifier assigned by Redpanda Cloud.
